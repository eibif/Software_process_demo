# 1.4 功能需求——考试管理

考试管理模块是连接“题库”与“成绩”的核心纽带，负责定义考试规则、执行考试过程监控以及处理试卷提交逻辑。

## 3.4.1 考试创建与信息配置（教师端）
| 属性         | 内容                                                         |
| :----------- | :----------------------------------------------------------- |
| **功能描述** | 教师通过该功能设定考试的基本属性、生效时间、答题时长及参与考生的范围。 |
| **前置条件** | 教师已登录系统；系统中已存在可选的题库资源。                 |
| **触发条件** | 教师点击“新建考试”按钮。                                     |
| **后置结果** | 生成一条“未发布”或“待开始”状态的考试记录，并在指定时间对学生可见。 |
| **主流程**   | 1. 教师输入考试名称、描述；<br>2. 设定考试起止时间（时间窗）和有效答题时长（如 60 分钟）；<br>3. 选择参与考试的学生名单或所属班级；<br>4. 设定考试防作弊开关（如允许切屏次数、随机乱序等）。 |
| **异常处理** | 1. 时间冲突：结束时间早于开始时间，系统提示错误；<br>2. 考生为空：未指定考生即发布，系统弹出警告。 |
| **数据要求** | 输入字段：考试标题（必填）、时长（分钟）、考生ID列表、考试状态位。 |

## 3.4.2 组卷规则定义（教师端）
| 属性         | 内容                                                         |
| :----------- | :----------------------------------------------------------- |
| **功能描述** | 教师定义试卷的题目构成方式，支持手动挑选和按规则随机抽取。   |
| **前置条件** | 考试基础信息已配置完成。                                     |
| **触发条件** | 教师进入“组卷设置”环节。                                     |
| **后置结果** | 关联考试与具体题目，锁定试卷总分。                           |
| **主流程**   | 1. **方式A（手动）**：教师从题库勾选题目，设定每题分值；<br>2. **方式B（随机）**：教师设定题型比例（如单选 10 题、多选 5 题）、难度系数及每题分值，系统从对应分类中随机抓取题目；<br>3. 系统自动计算并显示试卷总分。 |
| **依赖关系** | 上游依赖：M2 题库管理（题目分类及分值）；下游影响：M4 自动阅卷。 |
| **数据要求** | 组卷模式标识（0-手动/1-随机）、题目ID关联表、各题型分值配置。 |

## 3.4.3 在线考试过程控制（学生端）
| 属性         | 内容                                                         |
| :----------- | :----------------------------------------------------------- |
| **功能描述** | 学生在考试进行中，系统需实时维护考试状态、同步倒计时并执行防作弊监控。 |
| **前置条件** | 学生已登录；当前时间处于考试允许进入的时间窗内；学生在该考试名单中。 |
| **触发条件** | 学生点击“开始考试”。                                         |
| **后置结果** | 记录进入考场时间，初始化答题快照。                           |
| **主流程**   | 1. 系统校验学生是否有考试权限及是否重复进入（防多开）；<br>2. 加载试卷内容，启动前端渲染（一题一页或整卷模式）；<br>3. 开启服务器端同步倒计时；<br>4. **自动保存**：每间隔 30 秒或切换题目时，静默上传当前已选答案至后端缓存。 |
| **异常处理** | 1. 网络异常：保存失败时，前端提示“正在重连”，并在本地暂存答案；<br>2. 异常切屏：检测到焦点丢失，系统记录异常次数并向学生弹出警告。 |
| **数据要求** | 剩余秒数（实时同步）、当前作答 JSON 快照、切屏次数计数器。   |

## 3.4.4 交卷与状态锁定（系统/学生端）
| 属性         | 内容                                                         |
| :----------- | :----------------------------------------------------------- |
| **功能描述** | 处理学生主动交卷或系统由于时间截止、异常行为导致的强制收卷。 |
| **前置条件** | 考试处于“进行中”状态。                                       |
| **触发条件** | 情况A：学生点击“提交试卷”；情况B：倒计时归零。               |
| **后置结果** | 考试状态变更为“已提交”，锁定答案不可修改，触发 M4 自动阅卷模块。 |
| **主流程**   | 1. 检查是否有未完成题目，若主动交卷则弹出二次确认框；<br>2. 封存最终答案数据，并记录交卷时间戳；<br>3. 调用阅卷接口，将学生状态标记为“待出分”或“已完成”；<br>4. 跳转至考试结束反馈页面。 |
| **异常处理** | 1. 强制提交失败：若网络彻底中断导致自动交卷请求失败，学生重新联网登录后，系统应立即尝试同步本地最后的缓存数据并强制收卷。 |
| **依赖关系** | 核心依赖：非功能需求中的“数据一致性”与“并发控制”。           |

---

### 特别说明：
1.  **状态机的一致性**：考试管理的核心是一个严谨的状态机（未开始 -> 进行中 -> 已交卷 -> 已阅卷）。Agent 在设计数据库和后端逻辑时，必须确保“已交卷”状态后的任何更新请求（Update Answer）都被后端拒绝。
2.  **倒计时的可信度**：虽然前端有倒计时显示，但 Agent 在实现时，必须以**服务器端的时间戳差值**作为判定是否超时的唯一标准，防止学生通过修改本地系统时间作弊。
3.  **零编码交付的测试重点**：考试管理模块的集成测试应重点模拟“多人同时点击交卷”和“考试中途断网后恢复”的场景，验证 Agent 生成的并发处理代码和异常捕获逻辑是否稳健。
