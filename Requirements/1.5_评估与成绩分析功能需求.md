## 3. 需求规格说明

### 需求1：客观题自动批阅

| 项目 | 内容 |
|------|------|
| **需求标题** | 客观题自动批阅 |
| **功能描述** | 该功能允许系统在考试结束后，自动对学生提交的客观题（单选题、多选题、判断题）答案进行批阅，对比标准答案后计算得分，解决教师逐题批阅客观题耗时耗力的问题。 |
| **前置条件** | 1. 考试已结束且状态为“已截止”；<br>2. 试卷中已配置客观题的标准答案和每道题的分值；<br>3. 学生已提交答卷。 |
| **触发条件** | 1. 考试截止时间到达后系统自动触发；<br>2. 教师手动点击“开始批阅”按钮。 |
| **后置结果** | 1. 系统生成每道客观题的得分记录；<br>2. 系统更新并单独存储学生的客观题总分；<br>3. 试卷状态更新为“客观题已批阅”。 |
| **依赖关系** | 1. 上游依赖：考试管理模块（提供试卷结构、标准答案）、答题记录模块（提供学生答案）；<br>2. 下游影响：成绩合成模块。 |
| **主流程** | 1. 系统获取指定考试的所有学生答卷；<br>2. 系统逐题读取学生答案与标准答案；<br>3. 系统进行答案比对（单选题完全匹配计满分，否则0分；多选题全对计满分，漏选按比例给分，错选0分；判断题完全匹配计满分）；<br>4. 系统累加客观题总分；<br>5. 系统保存客观题每题得分和客观题总分至数据库；<br>6. 系统返回批阅完成提示。 |
| **异常处理** | 1. **无数据**：没有学生提交答卷时，系统提示“无答卷需批阅”；<br>2. **答案缺失**：某题未配置标准答案时，系统跳过该题并记录异常日志，提示教师补充答案；<br>3. **参数非法**：学生答案包含非法字符时，该题按0分处理并标记异常；<br>4. **超时**：大批量批阅超时时，系统支持断点续批；<br>5. **重复提交**：已批阅的答卷不会重复批阅。 |
| **数据要求** | 1. **输入字段**：考试ID（必填，整数，>0）、答卷ID列表（可选，为空时批阅全部）；<br>2. **输出字段**：批阅记录数（整数）、成功/失败状态（布尔值）、异常列表（数组）、客观题总分（浮点数）；<br>3. **内部规则**：多选题漏选得分=题目分值×(正确选项数/总选项数)，结果四舍五入保留1位小数。 |

### 需求2：主观题人工评阅

| 项目 | 内容 |
|------|------|
| **需求标题** | 主观题人工评阅 |
| **功能描述** | 该功能允许教师对主观题（简答题、论述题、计算题等）进行人工评分和评语录入，解决主观题无法自动判定的问题，同时支持多教师分题评阅和仲裁机制。 |
| **前置条件** | 1. 教师已登录且具有评阅权限；<br>2. 试卷中主观题已配置满分值。 <br>3.评卷任务下发给该教师|
| **触发条件** | 教师点击“评阅主观题”按钮并选择待评阅答卷。 |
| **后置结果** | 1. 生成每道主观题得分记录；<br>2. 主观题总分被单独存储（不含客观题）；<br>3. 教师评语被关联到答卷；<br>4. 试卷状态更新为“客观题已批阅”。 |
| **依赖关系** | 1. 上游依赖：答卷模块（提供学生答案）、题库模块（提供主观题满分配置）；<br>2. 下游影响：成绩合成模块。 |
| **主流程** | 1. 教师选择考试和班级；<br>2. 系统展示待评阅的学生列表及主观题作答内容；<br>3. 教师输入每题得分（0~满分值）和评语（可选）；<br>4. 系统校验得分是否在有效范围内；<br>5. 系统累加主观题；<br>6.系统保存z主观题每题得分、评语和主观题总分至数据库；<br>6. 系统返回批阅完成提示。 |
| **异常处理** | 1. **无权限**：非评阅教师或角色不符时，系统提示“无评阅权限”；2. **无数据**：无可评阅答卷时，提示“暂无需评阅的答卷”；3. **参数非法**：得分超出0~满分值范围时，提示“得分无效，请重新输入”；4. **重复提交**：同一题已评分时，系统提示“该题已评分，是否覆盖”；5. **仲裁场景**：两位教师评分差异超过阈值时，系统标记为“待仲裁”，提示仲裁教师介入。 |
| **数据要求** | 1. **输入字段**：答卷ID（必填，整数）、题目ID（必填，整数）、得分（必填，浮点数，范围[0,题目满分]）、评语（可选，字符串，最大500字符）；<br>2. **输出字段**：保存成功状态（布尔值）、主观题总分（浮点数）；<br>3. **评语格式**：支持纯文本，禁止HTML/JS代码。 |

### 需求4：教师端成绩分析

| 项目 | 内容 |
|------|------|
| **需求标题** | 成绩分析 |
| **功能描述** | 该功能允许教师对考试成绩进行统计分析和对比分析，包括单次考试的统计指标（平均分、最高分、最低分、标准差、及格率、优秀率、分数段分布等）以及不同班级或不同考试的成绩对比（横向、纵向），以图表和数据形式直观展示，帮助教师全面掌握班级和学生的成绩情况及变化趋势。 |
| **前置条件** | 1. 教师已登录且具有分析查看权限；<br>2. 至少有一个已批阅完成的考试；<br>3. 对比分析需至少有两个对比对象（班级或考试）。 |
| **触发条件** | 教师进入“成绩分析”页面，选择统计或对比分析功能。 |
| **后置结果** | 1. 系统展示统计分析图表（如平均分、分数段分布等）；<br>2. 系统展示对比分析图表（如班级/考试对比柱状图、趋势折线图等）；<br>3. 系统支持切换分析对象和分析类型。 |
| **依赖关系** | 上游依赖：成绩合成模块（提供所有学生成绩数据）。 |
| **主流程** | 1. 教师选择分析类型（统计分析/对比分析）；<br>2. 若统计分析，选择考试和班级，系统查询该班级所有有效成绩，计算统计指标（平均分、最高分、最低分、中位数、标准差、及格率、优秀率、分数段统计等），生成统计图表和成绩列表；<br>3. 若对比分析，选择对比类型（班级对比/历次对比），选择对比对象，系统查询各对象的统计数据，生成对比图表（柱状图、折线图等）和对比明细表，自动计算变化量； |
| **异常处理** | 1. **无权限**：教师无权查看时，提示“无权限”；<br>2. **无数据**：无有效成绩时，显示“暂无成绩数据”；<br>3. **对比对象不足**：对比分析需至少2个对象，不足时提示；<br>4. **计算异常**：数据量过大导致超时时，提示“数据加载超时，请缩小查询范围”；<br>5. **图表渲染失败**：系统提示“图表生成失败，请重试”；<br>6. **班级无学生**：提示“该班级无参考学生”。 |
| **数据要求** | 1. **输入字段**：考试ID（必填）、班级ID（统计分析时必填）、对比类型（枚举：class/exam，对比分析时必填）、对比对象ID列表（数组，对比分析时必填）；<br>2. **输出字段**：平均分（浮点数，保留2位）、最高分（浮点数）、最低分（浮点数）、标准差（浮点数）、及格率（百分比）、优秀率（百分比）、分数段统计（Map<区间,人数>）、对比数据集（含各对象名称、平均分、及格率、优秀率）、变化量（浮点数）、图表Base64（可选）；<br>3. **分数段默认区间**：[0,60)、[60,70)、[70,80)、[80,90)、[90,100]；<br>4. **图表格式**：支持PNG/JPG下载。 |

### 需求5：试题难度与区分度分析

| 项目 | 内容 |
|------|------|
| **需求标题** | 试题难度与区分度分析 |
| **功能描述** | 该功能允许用户查看每道试题的难度系数（通过率）和区分度（高分组与低分组的得分差异），帮助教师优化试卷、帮助学生了解各题难易程度和自身表现。 |
| **前置条件** | 1. 用户已登录；<br>2. 考试已批阅完成且参与人数≥30（区分度计算有统计学意义）；<br>3. 试卷结构数据完整。 |
| **触发条件** | 用户在成绩分析或成绩查询页面点击“试题质量分析”或“查看试题难度”。 |
| **后置结果** | 1. 系统展示每道题的难度系数；<br>2. 系统展示每道题的区分度指标；<br>3. 系统标记异常试题（过难、过易、区分度差）。 |
| **依赖关系** | 上游依赖：成绩合成模块（提供每题得分数据）、答题记录模块（提供各题答题详情）。 |
| **主流程** | 1. 用户选择考试；<br>2. 系统获取所有学生各题得分明细；<br>3. 系统计算每题难度系数 = 该题平均分 / 该题满分值；<br>4. 系统按总成绩排序，取前27%为高分组、后27%为低分组；<br>5. 系统计算每题区分度 = (高分组平均分 - 低分组平均分) / 题目满分值；<br>6. 系统标记：难度<0.3为“偏难”，>0.8为“偏易”；区分度<0.2为“较差”，0.2~0.3为“尚可”，≥0.3为“良好”；<br>7. 系统展示试题分析列表，教师端支持导出分析报告。 |
| **异常处理** | 1. **样本不足**：参考人数<30时，系统提示“样本量不足，区分度结果仅供参考”；<br>2. **无数据**：无答题数据时，提示“无有效数据进行分析”；<br>3. **计算异常**：题目满分值为0时，跳过该题并记录日志。 |
| **数据要求** | 1. **输入字段**：考试ID（必填）；<br>2. **输出字段**：题目ID、题目内容（简略）、难度系数（0~1，保留2位）、区分度（-1~1，保留2位）、质量评价标签（字符串）；<br>3. **区分度评价标准**：≥0.35为优秀，0.25~0.35为良好，0.15~0.25为尚可，<0.15为差。 |

| **异常处理** | 1. **无权限/未登录**：提示“请先登录或无权下载”；<br>2. **成绩未发布**：提示“成绩尚未公布”；<br>3. **文件生成失败**：提示“成绩单生成失败，请重试”。 |
| **数据要求** | 1.**输入字段**：学生ID（由系统从会话获取）、考试ID、导出格式（可选）；<br>2.**输出字段**：下载URL、文件大小；<br>3.**文件要求**：PDF优先，文件名格式为“成绩单_学生姓名_考试名称_日期.pdf”。 |

### 需求8：成绩复核与申诉处理

| 项目 | 内容 |
|------|------|
| **需求标题** | 成绩复核与申诉处理 |
| **功能描述** | 该功能允许学生对已发布的考试成绩提出复核申请，说明申诉理由；教师或管理员接收申请后，调阅答卷进行复核，决定是否调整成绩。解决学生对成绩有异议时无法正式提出申诉、流程不透明的问题，保障考试的公平公正。 |
| **前置条件** | 1. 考试状态为“已发布成绩”；<br>2. 学生已登录系统；<br>3. 该学生参加了对应考试；<br>4. 申诉时间在规定的申诉期限内（如成绩发布后7天内）。 |
| **触发条件** | 学生在成绩查询页面点击“申请复核”按钮，填写申诉信息后提交。 |
| **后置结果** | 1. 系统生成复核申请单，状态为“待处理”；<br>2. 教师收到待复核通知；<br>3. 复核完成后，系统更新成绩（若调整）并通知学生；<br>4. 复核记录永久保存。 |
| **依赖关系** | 1.上游依赖：成绩查询模块（提供学生成绩信息）、主观题评阅模块（提供答卷原始记录）；<br>2.下游影响：成绩统计模块（复核调整后需重新计算统计指标）。 |
| **主流程** | 1. 学生在“我的成绩”页面选择目标考试；<br>2. 系统检查是否在申诉期内且在申诉次数限制内（每科最多1次）；<br>3. 学生点击“申请复核”按钮；<br>4. 系统弹出复核申请表单；<br>5. 学生选择需复核的题目（可多选）、填写申诉理由（必填，至少10字符）、上传证明材料（可选）；<br>6. 系统校验表单完整性；<br>7. 系统生成复核申请单（申请编号、提交时间、状态=待处理）；<br>8. 系统向对应教师/管理员发送待复核通知；<br>9. 教师登录后进入“复核管理”页面，查看待复核列表；<br>10. 教师点击某条申请，调阅学生原始答卷；<br>11. 教师判定原成绩是否有误；<br>12. 教师输入复核结果（成绩维持不变/成绩调整），若调整则输入新得分；<br>13. 系统校验新得分是否在题目满分范围内；<br>14. 系统保存复核结果，更新申请单状态为“已处理”；<br>15. 若成绩有调整，系统自动更新学生该题得分、主客观分、总成绩；<br>16. 系统发送复核结果通知给学生；<br>17. 系统记录成绩变更日志（原分数、新分数、操作人、操作时间）；<br>18. 系统触发关联模块（如成绩统计）重新计算相关指标。 |
| **异常处理** | 1. **无权限**：非本人或非授权教师操作时，提示“无权限执行此操作”；<br>2. **超时/过期**：超出申诉期限时，提示“申诉已截止，无法提交”；<br>3. **重复提交**：该生已对该考试提交过复核申请且尚未完成处理时，提示“已有正在处理中的复核申请，请勿重复提交”；<br>4. **参数非法**：申诉理由过短（<10字符）时，提示“申诉理由需至少10个字符”；新得分超出范围时，提示“得分无效，请输入0~题目满分之间的数值”；<br>5. **无数据**：申请单不存在时，提示“复核申请不存在或已被删除”；<br>6. **并发冲突**：同一申请单被多位教师同时处理时，系统采用乐观锁，后提交者提示“该申请已被处理，请刷新页面”；<br>7. **撤销/回滚**：成绩调整后，若发现调整有误，管理员可回滚至原成绩并保留操作日志。 |
| **数据要求** | 1.**输入字段**：考试ID（必填，整数）、申请题目ID列表（必填，数组）、申诉理由（必填，字符串，10-500字符）、证明材料（可选，支持jpg/png/pdf，单文件≤5MB）；<br>2.**学生端输出字段**：申请单ID（整数）、申请编号（字符串，格式：RF+日期+序号）、状态（枚举：待处理/已处理/已驳回）、复核结果（字符串）。<br>3.**教师端输出字段**：原得分（浮点数）、复核后得分（浮点数，默认等于原分）。<br>4.**日志字段**：操作人ID、操作时间、变更前值、变更后值、变更原因。<br>5.**申诉期配置**：默认成绩发布后7天内，管理员可配置（1-30天）。 |
