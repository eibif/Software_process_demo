# 课程大作业 Demo 整体规划

## 1. 当前阶段目标

当前阶段交付内容为课程大作业的前端 Demo，重点是把系统的角色入口、页面骨架、核心流程和模块边界先搭起来，便于小组后续继续扩展题库管理、考试管理和成绩分析等功能。

## 2. Demo 总体定位

本项目定位为在线考试平台原型，当前首批 Demo 主要承担两层职责：

- 作为系统统一身份入口，完成登录、注册、角色区分和会话控制；
- 作为后续模块的公共基础壳层，提供统一导航、页面结构、状态上下文和访问控制。

## 3. 当前优先模块

本轮优先完成 `1.2 用户管理`，因为它是其余模块的基础设施：

- 题库管理依赖教师 / 管理员身份与权限控制；
- 考试管理依赖学生范围、会话和账号状态；
- 成绩分析依赖用户身份与可见范围；
- 系统公共 UI 壳层也应当先在用户管理模块中落地。

## 4. 当前 Demo 中的角色设计

### 4.1 学生

- 使用统一入口登录；
- 进入个人工作台；
- 只能访问自己的信息与后续个人相关功能。

### 4.2 教师

- 进入教师工作区；
- 在用户管理模块中只查看本人及所属学生；
- 负责学生账号维护，不承担全局角色配置。

### 4.3 管理员

- 进入管理员工作区；
- 管理全平台用户、角色与账号状态；
- 与教师角色明确分离。

## 5. 本轮实现策略

- 采用纯前端 Demo，不接入真实后端和数据库。
- 用本地存储模拟用户、日志和会话状态。
- 当前 Demo 中用户新增 / 编辑采用单角色分配。
- 页面结构、角色边界和交互流程尽量按照真实系统组织。
- 保持模块拆分清晰，便于组内其他同学直接扩展。

## 6. 当前已经形成的公共基础

- 统一登录 / 注册 / 找回密码入口
- 统一后台布局与导航
- 统一角色上下文与访问控制逻辑
- 统一用户数据结构
- 路由守卫与无权限提示
- 会话冲突演示机制

## 7. 后续模块接入建议

### 7.1 题库管理

复用当前后台壳层、教师角色、页面卡片和表格结构。

### 7.2 考试管理

复用用户范围、会话控制和学生名单字段。

### 7.3 成绩分析

复用角色可见范围、用户信息与审计日志基础。

## 8. 当前阶段结论

`1.2 用户管理` 不是孤立页面，而是整个在线考试系统的入口层与基础层。当前 Demo 的目标是先把这层做稳定、做清晰、做可扩展，再在此基础上继续接入后续功能模块。
